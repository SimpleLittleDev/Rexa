# Rexa MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a runnable TypeScript foundation for Rexa with LLM routing, CLI detection, tools, memory, storage, sub-agents, setup wizard, docs, and tests.

**Architecture:** The MVP uses config-driven provider/model selection and a safe local fallback provider. Core modules communicate through strict interfaces so real providers, storage engines, and chat surfaces can be added without rewriting the agent loop.

**Tech Stack:** Node.js, TypeScript, Vitest, optional Playwright, optional SQLite/PostgreSQL, optional Telegram/WebSocket.
